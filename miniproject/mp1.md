# Mini Project 1 Competency Claim

This project demonstrates **C3 (Data cleaning and file handling)** because I programmatically collected data from PokéAPI, transformed nested JSON into a structured pandas DataFrame, and engineered analysis-ready fields such as `total_stats` and `iconic_group` for later comparisons. I also handled expected missingness (for example, `secondary_type` for single-type Pokémon) and documented why those nulls are structurally valid rather than data errors.

This work demonstrates **C5 (Data analysis with pandas)** through question-driven analysis using correlation matrices, type-level summaries, η²-style effect sizes for stat–type association, a specialization index across mean stat profiles, and a permutation test with Cohen’s *d* for differences in `total_stats` across Pokédex ID bands. I used these methods to connect body metrics to speed, rank how tightly each stat aligns with primary type, and test whether “early ID” species differ in overall power from later entries.

This project demonstrates **C6 (Data visualization)** because Section 4 pairs each research question with a purpose-built Plotly figure: a log-scaled **scatter** of weight versus speed (with height as size and type as color) to surface relationship shape and cross-type variation; a horizontal **bar chart** of η² by base stat to show which stat dimension is most type-linked; and a **box plot** of total stats by ID-based iconicity groups with all points shown for transparency. Each chart uses a finding-oriented title, labeled axes, and an accompanying chart-rationale note.

This project demonstrates **C7 (Critical evaluation and professional judgment)** because I interpreted results with explicit assumptions (Pokédex ID as a proxy for iconicity), separated statistical patterns from causal claims about popularity, and flagged scope limits (single generation, base stats only, coarse physical fields). Section 5 ties those threads together so a reader knows what is supported by the evidence versus what would need richer data or modeling next.
